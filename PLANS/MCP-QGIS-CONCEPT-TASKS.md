# MCP + QGIS: Task Plan к предТЗ (проект 4114)

Версия: 0.1  
Дата старта: 2026-02-11  
Основание: анализ `4114/QGIS/MCP-QGIS-CONCEPT.md`

## Как пользоваться этим планом

- Отмечайте задачи чекбоксами по мере выполнения.
- Блок считается закрытым, когда закрыты все задачи блока.
- Статус блока:
- `[ ] Не начат`
- `[~] В работе`
- `[x] Завершен`

---

## Блок 1. Формальный контракт MCP-инструментов (Критично)

Что: описать строгие контракты всех MCP tools (входы/выходы/ошибки).  
Зачем: без контракта невозможно стабильно реализовать сервер и клиентов.  
Где: `4114/QGIS/MCP-QGIS-CONCEPT.md`, новый раздел `API Contract`; отдельный файл `4114/QGIS/MCP-TOOLS-SCHEMA.md` (или JSON Schema каталог).  
Когда: Спринт 1, День 1-2.  
Что ожидаем: полная спецификация tools с обязательными полями и единой моделью ошибок.

Таски:
- [x] Зафиксировать список MVP tools (12 шт. из концепта) и их назначение.
- [x] Для каждого tool описать request schema (required/optional, типы, диапазоны).
- [x] Для каждого tool описать response schema (успех, предупреждения, ошибки).
- [x] Ввести единый формат ошибок: `code`, `message`, `details`, `retryable`.
- [x] Ввести версию контракта `api_version` и правило совместимости.
- [x] Добавить примеры валидных и невалидных запросов.

Таски:
- [x] Описать модель авторизации (кто может делать какие операции).
- [x] Ввести уровни доступа: `read-only`, `edit`, `admin`.
- [x] Ограничить `execute_code`: default deny + явный allowlist.
- [x] Зафиксировать allowlist Processing-алгоритмов по профилю проекта.
- [x] Описать аудит: кто/когда/что сделал, hash плана и hash результата.
- [x] Описать политику хранения секретов и конфигураций.

Где: новый файл `4114/QGIS/MCP-HITL.md`.  
Когда: Спринт 3, День 1.  
Что ожидаем: прозрачная политика ручного контроля.

Таски:
- [x] Определить список “опасных” операций (массовое изменение, удаление, CRS override и т.д.).
- [x] Для каждой опасной операции ввести обязательный preview + explicit confirm.
- [x] Описать формат карточки подтверждения (что изменится, какие риски, как откатить).
- [x] Ввести уровни автоприменения: `auto`, `confirm`, `forbidden`.
